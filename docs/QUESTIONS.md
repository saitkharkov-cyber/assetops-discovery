# QUESTIONS

## Назначение

Документ содержит все открытые вопросы проекта.

Каждый вопрос должен быть либо:

* отвечен;
* превращен в гипотезу;
* привести к принятию решения;
* закрыт как неактуальный.

Вопрос не должен существовать бесконечно.

---

# Статусы

Каждый вопрос имеет один из статусов:

* Open
* In Progress
* Answered
* Closed

---

# Правила

Каждый вопрос получает уникальный идентификатор.

Формат:

```text
Q-001
Q-002
Q-003
...
```

---

# Реестр вопросов

## Q-001

### Статус

In Progress

### Вопрос

Существует ли достаточно большой рынок компаний с проблемой учета мобильных активов?

### Почему это важно

Если рынок слишком мал, разработка SaaS не имеет смысла.

* исследование компаний;
* Customer Discovery;
* интервью.

# Правила обновления

После получения ответа необходимо:

* изменить статус вопроса;
* зафиксировать ответ;
* при необходимости:

  * обновить `ICP.md`;
  * обновить `HYPOTHESES.md`;
  * создать запись в `DECISIONS.md`;
  * обновить `PROJECT_MASTER_SUMMARY.md`.

---

# Основной принцип

Вопросы существуют не для хранения информации.

Они существуют для уменьшения неопределенности.

Если вопрос больше не влияет на развитие проекта, он должен быть закрыт.
