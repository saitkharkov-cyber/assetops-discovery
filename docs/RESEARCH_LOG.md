# RESEARCH_LOG

## Назначение

Хронологический журнал исследований.

Документ фиксирует ход Customer Discovery и развитие понимания рынка.

Каждая запись должна описывать одно завершенное исследование.

---

# Правила

Одна исследованная компания = одна запись в журнале.

Подробный анализ компании хранится отдельно в:

`research/companies/`

В журнале фиксируется только краткий результат исследования.

---

# Формат записи

---

## YYYY-MM-DD

### Компания

Название компании

### Сайт

https://company.com

### Источник

Google Search / Рекомендация / Каталог / Другое

### Что обнаружено

Краткий список фактов.

Например:

* собственная электролаборатория;
* промышленный электромонтаж;
* высоковольтные работы;
* несколько типов инженерных услуг.

### Что НЕ обнаружено

Факты, отсутствие которых важно зафиксировать.

Например:

* не найдено подтверждение нескольких бригад;
* не найдено подтверждение собственного штата;
* отсутствуют вакансии.

### Предварительная оценка ICP

Высокая / Средняя / Низкая

### Какие гипотезы затронуты

Например:

* H-001
* H-002

### Итог

Краткий вывод одним абзацем.

---

# Правила ведения журнала

Записываются только:

* факты;
* наблюдения;
* выводы.

Не записываются:

* идеи продукта;
* технические решения;
* архитектура SaaS.

Для этого существуют другие документы.

---

# Связанные документы

После завершения исследования при необходимости обновляются:

* `PROJECT_MASTER_SUMMARY.md`
* `ICP.md`
* `HYPOTHESES.md`
* `DECISIONS.md`

Если исследование не влияет ни на один из этих документов, это тоже считается нормальным результатом.

---

# Цель журнала

Через несколько месяцев журнал должен позволять ответить на вопросы:

* Какие компании уже исследованы?
* Какие закономерности были обнаружены?
* Какие гипотезы подтвердились?
* Почему изменялся ICP?
* Как эволюционировало понимание рынка?

Журнал является историей исследования, а не историей переписки.
